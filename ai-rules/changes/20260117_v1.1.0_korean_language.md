## 변경 동기 (Why)
프로젝트 문서의 일관성과 협업 효율성을 높이기 위해 모든 산출물의 기본 언어를 한글로 통일. 동시에 LLM 토큰 효율성과 국제 표준 준수를 위해 CHANGELOG는 영어 메인 + 한글 병기 전략 채택.

## 변경 내용 (What)

### 1. core_principles.md에 원칙 추가
**5. Korean Language (한글 기본)** 섹션 신규 추가:
- 모든 프로젝트 문서(회의록, README, 설계 문서, 코드 주석)는 한글로 작성
- CHANGELOG는 영어 메인 + 한글 병기
  - 형식: `- [Feature] Add new component ([기능] 새 컴포넌트 추가)`
  - LLM 토큰 효율성을 위해 영어를 먼저 배치
- 예외 조항:
  1. 오픈소스 기여 및 외부 협업 문서
  2. 인프라 코드 및 설정 파일 (Dockerfile, YAML, Terraform 등)
  3. 학술 출판물 (논문, 국제 학회 발표)
  4. 국제 협업 프로젝트

### 2. 페르소나 검토 프로세스 통과
- 10년차+ 베테랑 페르소나 4명 (개발자, 연구원, 아키텍트, PM) 참여
- 3라운드 교차 토론 진행
  - 라운드 1: 독립 의견 제시 (조건부 찬성/찬성)
  - 라운드 2: 반박 및 보완 의견 (CHANGELOG 이중 언어 전략 합의)
  - 라운드 3: 최종 입장 정리 (모두 조건부 승인 이상)
- PM 최종 보고서: 조건부 승인 권고

### 3. 워크플로우 업데이트
`/.agent/workflows/update-ai-rules.md` 수정:
- CHANGELOG 작성 시 영어 메인 + 한글 병기 형식 안내 추가
- LLM 토큰 효율성 고려 사항 명시

- **CI/CD**:
