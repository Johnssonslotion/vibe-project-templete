# AI Rules Change Detail: v1.8.0 - Workflow Governance & README Integration

**버전**: v1.8.0  
**날짜**: 2026-01-17  
**변경 주체**: 워크플로우 거버넌스 및 가시성 강화  
**승인자**: 사용자  
**회의록**: [20260117_workflows_v1.8.0.md](../meeting_minutes/workflows/20260117_workflows_v1.8.0.md)

---

## 변경 동기 (Why)
거버넌스 워크플로우를 시스템 폴더(`.agent/`)에서 사용자 관리 폴더(`ai-rules/`)로 이동하여 거버넌스의 주권과 투명성을 확보함. 또한 프로젝트의 첫 인상인 `README.md`에 워크플로우 사용자를 위한 가이드를 통합하여 접근성을 극대화함.

## 변경 내용 (What)

### 1. 워크플로우 위치 이동 및 체계화
- **경로 이동**: `.agent/workflows/` -> `ai-rules/workflows/`.
- **전용 히스토리**: `ai-rules/workflows/HISTORY.md`를 통해 워크플로우별 진화를 개별 추적.
- **메타 거버넌스**: 워크플로우 자체의 품질을 통제하는 `manage-workflows.md` 도입.

### 2. 회의록 체계 세분화 (Reorganization)
- **거버넌스 영역**: `ai-rules/meeting_minutes/` 하위를 `rules/`(원칙)와 `workflows/`(절차)로 분리하여 합의 근거의 전문성을 높임.
- **문서/일반 영역**: `docs/meeting_minutes/general/`로 이동하여 거버넌스와 일반 프로젝트 문서를 명확히 격리.

### 3. 가시성 강화 (README)
- `README.md`에 주요 워크플로우 요약표를 추가하여 AI 에이전트와 사용자가 지시 없이도 사용 가능한 도구를 쉽게 인지하도록 함.

### 3. 통제 강화
- `Documentation Expert` 페르소나가 모든 워크플로우의 가독성과 표준 준수를 검토하도록 프로세스에 명문화.

## 영향 범위 (Impact)
- **사용자**: 프로젝트의 규칙과 실행 도구를 한 곳(`ai-rules/`)에서 일관되게 관리할 수 있음.
- **AI 에이전트**: 명확하게 모듈화된 워크플로우와 README의 인덱스를 통해 수행 정확도가 향상됨.
