## 변경 동기 (Why)
프로젝트 시작 시점에 AI 에이전트가 준수해야 할 기본 규칙과 프로세스를 명확히 정의하여, 개발 및 연구 활동의 일관성과 추적성을 확보하기 위함.

## 변경 내용 (What)

### 1. 디렉터리 구조 수립
```
ai-rules/
├── index.md            # AI Rules 진입점
├── core_principles.md  # 핵심 원칙
├── HISTORY.md          # 변경 이력 인덱스
└── changes/            # 상세 변경 문서
```

### 2. 핵심 원칙 정의 (`core_principles.md`)
- **Document-First**: 코드 작성 전 문서화 우선
- **Explicit Approval**: 중요 변경 시 명시적 승인 필요
- **Version Control & Traceability**: Git + CHANGELOG.md 기록 필수
- **Stability & Quality**: main/develop 브랜치 안정성 보장

### 3. 관리 프로세스 정의 (`index.md`)
- 규칙 우선순위 시스템 (index → core → tasks → docs)
- 변경 권한 및 절차 (제안 → 합의 → 반영 → 검증)
- AI 에이전트 행동 지침

### 4. 자동화 메커니즘
- `.agent/rules/ai-rules.md` 자동 동기화 (복사 방식)
- `pre-commit` 훅을 통한 동기화 자동화

## 영향 범위 (Impact)
- **개발 워크플로우**: 모든 PR은 AI Rules 준수 여부 검증 필요

- **CI/CD**: 규칙 파일 변경 시 승인 프로세스 추가 예정
