# Spec-First 구현 워크플로우

실제 코드 작성 전 상세 설계(Specification)를 확정하고 승인받는 절차입니다.

---

## 1. 관련 원칙
- **Principle 7**: Spec-First (설계 우선)

## 2. 단계별 프로세스

### 1단계: 요구사항 구체화
- RFC 또는 백로그 티켓 내용을 바탕으로 기능 요구사항 나열.
- 사용자 시나리오 및 예외 케이스 정의.

### 2단계: 상세 명세서 작성
- `specification.md` 생성 (또는 전용 설계 문서 브랜치 내 생성).
- **포함 사항**: API 정의, 데이터 모델 스키마, UI Mockup, 인프라 요구사항.

### 3단계: 전문가 검토
- **System Architect**: 설계 패턴 및 데이터 흐름 검토.
- **Documentation Expert**: 명세의 가독성 및 이력 관리 적정성 검토.

### 4단계: 사용자 승인
- 명세서 내용을 사용자에게 공유하고 **명시적 승인** 획득.
- 승인 완료 후 구현 브랜치(`feat/` 또는 `fix/`) 생성 및 작업 시작.

- [ ] 사용자 승인 로그(메시지) 확인
